@startuml
title 결제 처리 (주문 생성 이후)
actor User
User -> 결제 : 결제 요청 (주문ID, 포인트, 쿠폰)

결제 -> 포인트 : 포인트 차감 요청
포인트 -> DB : 포인트 업데이트
DB --> 포인트 : 차감 완료

결제 -> 쿠폰 : 쿠폰 사용 처리
쿠폰 -> DB : 사용 처리 완료

결제 -> DB : 결제 정보 저장
DB --> 결제 : 저장 완료

결제 -> 주문 : 결제 완료 알림
주문 -> DB : 주문 상태 변경 (결제 완료)

결제 --> 외부 플랫폼 : 주문 정보 전송
외부 플랫폼 --> 결제 : 전송 완료

결제 --> User : 결제 완료 응답

@enduml
